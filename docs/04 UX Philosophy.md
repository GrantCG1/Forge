# Purpose

The UX Philosophy defines how Forge should feel to use.

It provides guidance for every interaction, screen, notification and workflow to ensure a consistent, trustworthy and human-centred experience.

Every design decision should support Forge's mission of reducing cognitive load while helping users live healthier, more balanced lives.

---

# User Experience Vision

Forge should feel like an experienced coach walking alongside the user through nature.

It should inspire confidence rather than dependence.

It should educate without overwhelming.

It should encourage without judging.

It should quietly support users in making better decisions while respecting their autonomy.

---

# Emotional Goals

Within the first few moments of opening Forge, users should feel:

- Relaxed.
- Motivated.
- Curious.

Forge should reduce stress rather than add to it.

---

# Progressive Disclosure

Forge should provide the right amount of information at the right time.

The default experience should be simple, clear and actionable.

Additional explanations, educational content and deeper insights should always be available, but never forced upon the user.

Users who wish to learn more should always be able to do so.

---

# Recommendation Philosophy

Every recommendation should answer one simple question:

> "If I didn't want to think today, what would you recommend?"

Recommendations should always include:

- A clear recommendation.
- A concise explanation.
- Optional deeper learning.

The recommendation is the starting point, never the only path.

---

# Educational Experience

Education should occur naturally through everyday interactions.

Rather than separating learning from doing, Forge should explain its reasoning as recommendations are made.

Users should understand:

- Why something was recommended.
- What alternatives exist.
- What trade-offs different choices involve.

---

# Tone of Voice

Forge should communicate like a calm mentor.

Communication should be:

- Clear.
- Concise.
- Respectful.
- Encouraging.
- Honest.

Forge should avoid:

- Excessive enthusiasm.
- Fear-based messaging.
- Guilt.
- Pressure.
- Information overload.

---

# Motivation

Forge should never attempt to motivate through guilt.

When users fall behind, Forge should:

- acknowledge reality,
- adapt to their current circumstances,
- offer encouragement,
- provide achievable next steps.

Every interaction should reinforce the belief that progress is always possible.

---

# Celebration

Forge celebrates meaningful progress.

Achievements should be recognised proportionally.

Small wins deserve acknowledgement.

Major milestones deserve celebration.

Enthusiasm should always be calibrated to the significance of the achievement.

---

# Trust

Forge earns trust gradually.

It does so by:

- remembering user preferences,
- learning from feedback,
- acknowledging mistakes,
- explaining recommendations,
- adapting over time,
- respecting user choices.

Trust is built through consistency rather than persuasion.

---

# Adaptability

Forge adapts to the user's life.

It should consider:

- available time,
- injuries,
- motivation,
- experience,
- goals,
- changing circumstances.

Adaptation should always remain transparent.

Users should understand why recommendations change.

---

# Onboarding Philosophy

Forge should begin with a small number of essential questions.

Reasonable assumptions may be made initially.

As users interact with Forge, recommendations should become increasingly personalised.

Users should always be able to adjust the level of challenge and recommendation style as their goals evolve.

---

# Simplicity

Beautiful interfaces are simple interfaces.

Every screen should present only the information necessary for the current decision.

Complexity should remain hidden until requested.

---

# Guiding Principles

Every interaction should answer:

- Does this reduce cognitive load?
- Does this educate?
- Does this preserve autonomy?
- Does this build trust?
- Does this feel calm?
- Does this encourage balanced living?

If the answer is no, the interaction should be reconsidered.

---

# Guiding Statement

> Forge should feel like an experienced coach who understands your journey, remembers where you've been, and quietly helps you take the next step.
