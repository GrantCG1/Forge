# Purpose

This document records significant product decisions made throughout the development of Forge.

Its purpose is to capture not only *what* was decided, but *why* it was decided.

The Decision Log provides historical context and ensures important reasoning is never lost.

If a future decision changes an earlier one, the original decision should remain in this document together with the reason for the change.

---

# Decision History

## 2026-07-08

### D001 — Documentation is the source of truth

**Decision**

Project documentation is considered the authoritative source for Forge.

AI conversations are used for brainstorming, discussion and refinement, but once a decision is agreed, it must be documented before it is considered official.

**Reason**

Conversations are difficult to search months later.

Documentation creates clarity, consistency and long-term maintainability.

---

### D002 — Build the blueprint before the application

**Decision**

Forge will be fully designed before software development begins.

**Reason**

Strong products are built from thoughtful design rather than evolving through code alone.

---

### D003 — Forge educates, adapts and assists

**Decision**

Every feature within Forge should support one or more of these three responsibilities.

**Reason**

These three concepts represent the core philosophy of the product.

---

### D004 — The user remains in control

**Decision**

Forge should make recommendations, not decisions.

The user always retains ownership of meaningful choices.

**Reason**

The goal is to build confidence and independence rather than dependence.

---

### D005 — Technology should reduce effort without reducing capability

**Decision**

Forge should reduce unnecessary cognitive load while continuing to develop the user's knowledge and skills.

**Reason**

Technology should make life easier without making people less capable.
